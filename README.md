# Hackathon Todo App

A full-stack todo application with AI chatbot integration, built across three phases using Spec-Driven Development.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 16 (App Router), TypeScript, Tailwind CSS, shadcn/ui |
| Backend | Python FastAPI, SQLModel ORM |
| Database | Neon Serverless PostgreSQL |
| Auth | Better Auth with JWT plugin (HS256) |
| AI/LLM | Cohere API (Command models) with tool use |
| Dev | docker-compose, Claude Code + Spec-Kit Plus |

## Phases

1. **Phase I** - Console todo app (completed)
2. **Phase II** - Full-stack web application with auth and CRUD (completed)
3. **Phase III** - AI chatbot with natural language task management via Cohere tool calling (current)

## Project Structure

```
hackathon-todo/
  frontend/          # Next.js app (port 3000)
  backend/           # FastAPI server (port 8000)
  specs/             # Feature specifications
  history/           # Prompt History Records
  .specify/          # Spec-Kit Plus config
  .spec-kit/         # Project phases config
  docker-compose.yml # Local dev orchestration
```

## Setup

### Prerequisites

- Node.js 18+
- Python 3.11+
- Neon PostgreSQL database

### Environment Variables

Copy the example files and fill in your values:

```bash
cp frontend/.env.example frontend/.env.local
cp backend/.env.example backend/.env
```

Required variables:
- `DATABASE_URL` - Neon PostgreSQL connection string
- `BETTER_AUTH_SECRET` / `JWT_SECRET_KEY` - Shared HS256 signing secret
- `CO_API_KEY` - Cohere API key (Phase 3)

### Frontend

```bash
cd frontend
npm install
npm run dev
```

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### Docker Compose

```bash
docker-compose up
```

Frontend at http://localhost:3000, backend at http://localhost:8000.

## Development Workflow

This project follows Spec-Driven Development (SDD):

1. Write/update spec (`specs/<feature>/spec.md`)
2. Generate plan (`/sp.plan`)
3. Break into tasks (`/sp.tasks`)
4. Implement via Claude Code (`/sp.implement`)

All code changes are generated by Claude Code. No manual coding permitted.
